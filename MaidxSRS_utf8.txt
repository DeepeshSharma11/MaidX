Introduction1.1 PurposeThe purpose of this document is to define the requirements for the MaidX system, a web/mobile platform that connects domestic workers (maids) with clients. It enables maids to create profiles and allows clients to hire them based on specific requirements and time slots.1.2 ScopeMaidX is an online service platform where:Maids can register and showcase their skills and availability. Clients can search, filter, and hire maids. Booking is based on time slots and requirements. The system includes a help desk and report management system. 2. Overall Description2.1 Product PerspectiveMaidX is a standalone application that can be deployed as:Web Application Mobile Application (Android/iOS) It acts as a mediator between service providers (maids) and service seekers (clients).2.2 User ClassesAdminMaid (Service Provider)Client (Service Consumer)2.3 Operating EnvironmentWeb browsers (Chrome, Edge, etc.) Android/iOS devices Backend server (Cloud-based) 3. Functional Requirements3.1 User Registration & LoginUsers (maids and clients) must register using: Name Phone number / Email Password Secure login authentication system. 3.2 Maid Profile ManagementMaids can: Create and update profile Add skills (cleaning, cooking, babysitting, etc.) Set availability (time slots) Set pricing per hour/day Upload identity proof 3.3 Client Profile ManagementClients can: Create and manage profiles Add address/location Define requirements (type of work, duration, timing) 3.4 Search & Hiring SystemClients can: Search maids based on: Location Skills Availability Ratings Book maids for specific time slots System should: Prevent double booking Show availability calendar 3.5 Booking ManagementClients can: View booking history Cancel or reschedule bookings Maids can: Accept or reject requests View upcoming bookings 3.6 Rating & Review SystemClients can rate maids after service Ratings affect maid visibility 3.7 Help Desk ModuleUsers can: Raise complaints/issues Track ticket status Admin can: Respond to queries Resolve issues 3.8 Report SectionSystem should generate reports such as: Total bookings Revenue reports User activity reports Complaint reports Admin dashboard for analytics 3.9 NotificationsEmail/SMS/App notifications for: Booking confirmation Cancellation Updates from help desk 4. Non-Functional Requirements4.1 PerformanceSystem should handle multiple users simultaneously Response time < 3 seconds 4.2 SecurityUser data encryption Secure authentication (JWT/OTP) Role-based access control 4.3 UsabilityUser-friendly interface Easy navigation Simple booking flow 4.4 Reliability99% uptime Backup and recovery system 4.5 ScalabilityShould support increasing number of users 5. System Features5.1 Admin PanelManage users (maids & clients) Monitor bookings Handle complaints Generate reports 5.2 Time Slot ManagementMaids define available slots Clients select preferred timing 6. External Interface Requirements6.1 User InterfaceResponsive UI Dashboard for each user type 6.2 Hardware InterfaceSmartphone / Computer 6.3 Software InterfaceDatabase (MySQL / MongoDB) Backend (Node.js / Django) Frontend (React / HTML/CSS) 7. ConstraintsInternet connection required Data privacy regulations must be followed 8. Future EnhancementsAI-based maid recommendations Real-time tracking Online payment integration Multi-language support 9. ConclusionMaidX aims to simplify the hiring process of domestic help by providing a reliable, secure, and efficient platform. It benefits both maids and clients by ensuring transparency, convenience, and accessibility.
